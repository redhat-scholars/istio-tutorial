Istio is supported in OpenShift by Red Hat OpenShift Service Mesh operator. This section assumes you have an OpenShift 4.9+ cluster and an account with administrator access.

The steps below provide a summary of the installation steps for Red Hat OpenShift Service Mesh. Detailed instructions can be found in the OpenShift Documentation.

Using Istio on OpenShift requires the installation of four operators, configuring a Mesh Control Plane, and enrolling namespaces in the mesh using a Mesh Member Roll. The required operators are:

  • OpenShift Elastic Search Operator

  • OpenShift distributed tracing platform (based on Jaeger)

  • Kiali Operator

  • OpenShift Service Mesh Operator

Install the operators by following these steps:

  1. Login to your OpenShift cluster’s console as user with administrator permissions, e.g the kubeadmin account.

  2. Choose the Administrator perspective from the side-menu.

  3. Expand the Operators section and select Operator Hub.

  4. Check the Red Hat box under the Source section to filter out community operators.

  5. Search for and install the OpenShift Elastic Search Operator. Accept the default installation parameters when prompted.

  6. Repeat the previous steps to install the following operators:

    • OpenShift distributed tracing platform

    • Kiali

    • OpenShift Service Mesh

It’s recommended, but optional, to install the Web Terminal operator to simplify running commands later in this guide. You can use the OpenShift (oc) CLI and kubectl directly from your development machine if you prefer. The OpenShift CLI can be downloaded using the Command line tools link from the help icon in the top-right of the OpenShift console.

Once complete select Installed Operators from the side-menu in the OpenShift console and verify the are listed as installed. Your list should be similar to the list shown in the following screenshot.

To configure Istio we need to create a namespace where the control plane components can be deployed. To create the istio-system namespace:

  1. Expand the Home section and select Projects.

  2. Click the Create Project button in the top-right of the screen. A modal will appear in the center of the screen.

  3. Enter istio-system in the Name and click Create in the modal.

You can now configure and deploy Istio:

  1. Expand the Operators section and select Installed Operators.

  2. Click the Istio Service Mesh Control Plane item in the Provided APIs column next to the Red Hat Service Mesh Operator

  3. Click the Create ServiceMeshControlPlane button.

  4. Leave the Istio configuration parameters with the default values except Proxy > Networking > Traffic Control > Outbound > Policy which needs to be changed to REGISTRY_ONLY to prevent traffic going outside of the mesh.

  5. Scroll down and click Create to create the Istio Control Plane.

  6. Click on the newly listed basic item in the ServiceMeshControlPlane list.

  7. Wait until all Istio components are shown as ready.

Lastly, enroll services from a namespace in the Mesh using the Istio Service Mesh Member Roll:

  1. Expand the Home section and select Projects.

  2. Click the Create Project button in the top-right of the screen. A modal will appear in the center of the screen.

  3. Enter tutorial in the Name and click Create in the modal.

  4. Select the Operators > Installed Operators from the side-menu.

  5. Select the Istio Service Mesh Member Roll option from the Provided APIs column.

  6. Click the Create ServiceMeshMemberRoll button.

  7. Choose the YAML View and add the tutorial namespace to the spec.members list:

    apiVersion: maistra.io/v1
kind: ServiceMeshMemberRoll
metadata:
  name: default
  namespace: istio-system
spec:
  members:
    - tutorial

  8. Scroll down and click the Create button.

Congratulations! You’ve deployed Istio and enrolled services deployed in the tutorial project in the mesh.
